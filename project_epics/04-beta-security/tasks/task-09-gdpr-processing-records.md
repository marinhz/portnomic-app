# Task 4.9 — GDPR: processing records and subprocessors

**Epic:** [04-beta-security](../epic.md)

---

## Objective

Document processing activities and subprocessors (LLM, email providers); DPAs/SCCs where applicable (EDD §13).

## Scope

- **Processing records:** Logs or document describing processing activities (what data, purpose, retention, safeguards); available for DPA and audits (EDD §13).
- **Subprocessors:** List LLM and email providers; document DPAs and Standard Contractual Clauses (SCCs) where required (EDD §13).
- **Data flow:** High-level data flow (ingest, parse, DA, storage, export) for transparency.
- Maintain and version this documentation; review when adding new processors.

## Acceptance criteria

- [ ] Processing records document is created and kept up to date; accessible for audits.
- [ ] Subprocessors are listed; DPAs/SCCs in place where applicable.
- [ ] Documentation is sufficient for DPA review and GDPR accountability (EDD §13).
